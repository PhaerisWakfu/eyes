# 安全说明

## 报告漏洞

若你发现与本仓库**代码**相关的安全问题（例如脚本注入、依赖漏洞的可利用路径），请通过 **GitHub Security Advisories**（仓库 → Security → Report a vulnerability）联系维护者；若该入口不可用，可开 **私有 Issue** 或邮件（在仓库维护者资料中可见时）说明，**请勿**在公开 Issue 中粘贴可利用细节。

本工具会抓取第三方网站与 RSS，运行 `npm run fetch` 即表示你信任网络来源与本地执行环境；这不属于本仓库「服务端」漏洞范畴，但若有**我们代码中**因处理不可信数据而导致的明确 RCE/路径穿越等，欢迎按上述渠道报告。

## 密钥与配置

- 永远不要把真实 `PH_TOKEN`、Cloudflare API 令牌、公众号密钥等写入仓库。
- 使用 `.env`（已列入 `.gitignore`）或 CI 密文存储。
