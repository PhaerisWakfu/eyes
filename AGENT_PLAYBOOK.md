# Eyes Agent Playbook — 数据增强工作流

本文档指导编程 Agent 对 Eyes 抓取的原始数据进行 AI 增强处理。

**目标读者**：内容面向国内用户，所有增强内容均使用中文，便于阅读和快速浏览。

## 展示规则

- **抓取**：每次可以多抓，raw / enriched 里保留全部数据。
- **页面展示**：前端**只展示当日最多 20 条**，按 `valueScore` 从高到低排序后取前 20 条。因此增强时请认真打价值分，分数直接决定哪些会出现在页面上。

## 工作流概述

1. **输入**：`data/raw/{date}.json` — 当日原始抓取数据（已做科技/技术/AI/商业模式过滤）
2. **输出**：`data/enriched/{date}.json` — 增强后数据
3. **操作**：为每条 item 补充**中文简介**、可选**标题翻译**、标签、价值评分

## 执行步骤

### Step 1: 读取原始数据

读取 `data/raw/{date}.json`，获取 `items` 数组。每条 item 结构：

```json
{
  "id": "hn-41234567",
  "title": "Show HN: A new approach to ...",
  "url": "https://example.com/article",
  "source": "hackernews",
  "score": 150,
  "comments": 45,
  "commentsUrl": "https://news.ycombinator.com/item?id=...",
  "description": "...",
  "createdAt": "2026-03-09T12:00:00Z"
}
```

### Step 2.0：两阶段流程（强制）

为避免在“20 名以后”浪费时间，增强分两阶段进行：

1. **阶段 A：先打价值分（快速）**  
   - 对当天 `items` **所有条目**先只做一件事：给出 `valueScore`（1-10）。  
   - 允许非常精简地判断：根据标题/描述、来源热度（如 HN score）、是否是实用工具/深度观点/重大商业动向等。  
   - **此阶段不要写长简介、不要翻译标题、不要找配图、不要打标签**，只要分数即可。

2. **阶段 B：只丰富 Top20（深入）**  
   - 按 `valueScore` 从高到低排序，取 **Top20**。  
   - **仅对 Top20** 生成完整字段：`summary_zh`、`title_zh`（可选）、`tags`、`image`（可选）、`insight`（可选）、`commentary`（点评，可选）。  
   - 输出到 `data/enriched/{date}.json` 时，**只包含 Top20 items**（这样前端展示与导出公众号都更高效）。

#### `summary_zh`（必填 — 简介 / 一段话概括）

- **每条必须有一段简介**，用**普罗大众能懂的话**介绍这条内容在讲什么、观点是什么、核心思想是什么
- **更深入，不走马观花**：中文，建议 **320～500 字**，要能让没读过原文的人也大致理解作者在说什么
- 若是一篇文章：介绍这篇文章的**主要观点和核心思想**，让没读过的人也能快速理解在讨论啥
- 若是一个项目/产品：说明它解决什么问题、有什么特点、对普通人/开发者有什么价值
- 站在**普通读者/大众视角**写，避免行话堆砌；若仅有标题无 description，根据标题和 url 推断并提炼

**分段与排版（重要）**：简介**必须分段**，不要整段挤在一起。用 `\n\n`（两个换行）分隔段落，建议 2～4 段为宜：
- **第一段**：开篇点题——背景、事件或主题是什么
- **中间段**：展开论述——核心观点、技术要点或产品特性
- **收尾段**：总结价值——对谁有意义、为何值得关注
- 每段 2～4 句话，段与段之间用空行隔开，便于阅读

#### `image`（选填 — 配图）

- 尽量给文章/项目补一张配图，用来提升阅读吸引力
- 优先顺序：原文 `og:image` / `twitter:image` → 文章头图 → 项目 Logo
- 只填图片 URL；如果找不到就留空，不要编造
- 需在构建前单独运行 `npm run download-images`：将 URL 下载到 `data/assets/{日期}/` 并更新为本地路径；构建时仅复制已有图片，不会重新抓取
- 若未填 `image`，脚本会尝试从 `item.url` 的页面抓取 og:image（部分站点无或有限制，可能失败）

#### `title_zh`（选填 — 标题中文翻译）

- 将英文标题翻译成中文，便于快速浏览
- 不超过 40 字，保持简洁
- 若标题已是中文或无需翻译可省略

#### `tags`（必填）

从以下标签体系中选择 1-3 个最相关的标签：

**技术方向：**
- `AI/ML` — 人工智能、机器学习、LLM 相关
- `Web` — Web 开发、前端、后端框架
- `基础设施` — 云计算、DevOps、数据库、系统架构
- `安全` — 信息安全、隐私、加密
- `开源` — 开源项目、开源社区
- `开发工具` — IDE、CLI、开发者效率工具
- `移动端` — iOS、Android 开发
- `数据` — 数据工程、分析、可视化
- `硬件` — 芯片、嵌入式、IoT

**商业方向：**
- `SaaS` — 软件即服务产品
- `商业模式` — 盈利模式、定价策略
- `创业` — 创业经验、融资、增长
- `独立开发` — 独立开发者、Side Project
- `产品设计` — UX/UI、产品策略

**其他：**
- `行业动态` — 科技行业新闻、政策法规
- `职业` — 技术招聘、职业发展
- `思考` — 技术哲学、深度思考文章

#### `valueScore`（必填）

1-10 整数，评估该条目对技术从业者的价值：

| 分数 | 标准 |
|------|------|
| 8-10 | **必读**：开创性技术/产品、深度洞察、改变行业格局的事件 |
| 5-7  | **值得关注**：实用工具发布、有启发的经验分享、重要技术更新 |
| 3-4  | **一般**：常规新闻、普通项目发布 |
| 1-2  | **低价值**：水帖、重复信息、过于小众 |

评分参考信号：
- HN score > 200 或 GitHub stars 增长快 → 通常 6+
- 有深度技术分析或独特商业视角 → 7+
- 纯展示无深度的内容 → 3-4

#### `insight`（选填，valueScore >= 7 时建议填写）

- 一句话，不超过 50 字
- 指出"这对谁有价值、为什么值得关注"
- 例："对 SaaS 创业者有参考价值——验证了 PLG 模式在垂直领域的可行性"

#### `commentary`（必填 — 点评）

- **以计算机科学家的视角**写一段短评，50～120 字
- **要有「人味」**：可以带点个人态度、幽默、感慨或质疑，避免冷冰冰的「客观总结」
- 可表达：为什么觉得有意思、哪里值得吐槽、联想到的类比、对技术路线的看法等
- 若内容过于常规或无感，可不填
- 例："这类「用 AI 绕过 copyleft」的案例会越来越多，法律还没跟上，社区规范先吵起来了——有意思的观察窗口。"

### Step 3: 输出增强数据

将增强后的数据写入 `data/enriched/{date}.json`，结构：

```json
{
  "date": "2026-03-09",
  "fetchedAt": "2026-03-09T10:00:00Z",
  "enrichedAt": "2026-03-09T15:00:00Z",
  "items": [
    {
      "id": "hn-41234567",
      "title": "Show HN: A new approach to ...",
      "url": "https://example.com/article",
      "source": "hackernews",
      "score": 150,
      "comments": 45,
      "commentsUrl": "https://news.ycombinator.com/item?id=...",
      "description": "...",
      "createdAt": "2026-03-09T12:00:00Z",
      "summary_zh": "一种新的数据库索引方法，通过学习查询模式自动优化索引结构，实测比传统B-tree快3倍。",
      "title_zh": "Show HN：一种新的数据库索引思路",
      "tags": ["基础设施", "AI/ML", "开源"],
      "valueScore": 8,
      "insight": "对数据库性能优化从业者有直接价值——可能改变索引设计的思路",
      "commentary": "这类「用 AI 绕过 copyleft」的案例会越来越多，法律还没跟上，社区规范先吵起来了——有意思的观察窗口。"
    }
  ]
}
```

## 质量要求

1. **面向国内用户**：summary_zh、title_zh 均用中文，表达自然，避免翻译腔
2. **不要编造信息**：无法判断时 summary_zh 写"需手动查看"，valueScore 给 5
3. **保持客观**：不夸大不贬低，基于可见信息评估
4. **保留原始字段**：增强是追加字段，不要删除或修改原始 title/url 等
5. **enrichedAt 时间戳**：使用当前时间的 ISO 格式

## 快捷指令

对 Agent 说以下任一句即可触发工作流：

- "按 AGENT_PLAYBOOK.md 增强 data/raw/2026-03-09.json"
- "增强今天的数据"
- "enrich latest"
